# Implementation Plan: Portfolio Website & CMS Dashboard

**Branch**: `001-portfolio-cms` | **Date**: 2026-05-12 | **Spec**: specs/001-portfolio-cms/spec.md
**Input**: Feature specification from `specs/001-portfolio-cms/spec.md`

## Summary

Build a personal portfolio website and admin CMS dashboard for Mohamed Elhabib
(Elhabib Portfolio). The public site features a glassmorphism-dark homepage with
hero, case studies grid, infrastructure/DevOps visual section, experience
timeline, bilingual blog, about page, and contact form. The CMS provides full
CRUD for all content entities, authentication with session security, image
uploads, content export, and SEO management. The platform follows the flat PHP
+ TailwindCSS + Alpine.js stack mandated by the constitution.

## Technical Context

**Language/Version**: PHP 8.1+
**Primary Dependencies**: Flat PHP (no frameworks), TailwindCSS v3, Alpine.js v3, PDO
**Storage**: SQLite
**Testing**: Manual (browser-based validation)
**Target Platform**: Linux VPS (Nginx)
**Project Type**: web-application (frontend + backend)
**Performance Goals**: Lighthouse 90+, <3s form submission, <1s language switch
**Constraints**: Flat PHP only, no frameworks, PDO prepared statements, CSRF
on all forms, <5MB uploads, glassmorphism performant on mobile
**Scale/Scope**: Single-admin portfolio, <1000 daily visitors, <50 projects,
<200 blog posts

Public entry point is `public/index.php` with Nginx handling URL rewriting
to route all requests through the front controller.
